# How to Add a Blog Post - Instructions for Founder

## Quick Overview
You can add blog posts directly through the GitHub website - no coding required!

---

## Step-by-Step Instructions

### **Option 1: Using GitHub Web Interface (Easiest)**

1. **Go to the repository:**
   - Visit: `https://github.com/gcharris/chateau-alaverdi-site`
   - Sign in with your GitHub account

2. **Navigate to the blog posts folder:**
   - Click on `blog` folder
   - Click on `_posts` folder

3. **Create a new post:**
   - Click the **"Add file"** button (top right)
   - Select **"Create new file"**

4. **Name your file:**
   - Use this format: `YYYY-MM-DD-title-of-post.md`
   - Example: `2025-11-15-harvest-update.md`
   - The `.md` extension is important!

5. **Write your content:**
   - Start with a title: `# Your Post Title`
   - Add your content below
   - Use simple formatting (see examples below)

6. **Save (Commit) your post:**
   - Scroll to bottom
   - Add a commit message: "Add blog post about [topic]"
   - Click **"Commit new file"**

7. **Your post is live!**
   - The website updates automatically in 1-2 minutes

## Writing Format (Markdown Basics)

### Headers
```
# Main Title (biggest)
## Subtitle
### Section heading
```

### Bold and Italic
```
**Bold text**
*Italic text*
```

### Lists
```
- Item one
- Item two
- Item three

1. First item
2. Second item
3. Third item
```

### Links
```
[Link text](https://website.com)
```

### Images
```
![Image description](photos/image-name.jpg)
```

## Blog Post Template

Copy and paste this template for each new post:

```markdown
# Your Post Title Here

**Date:** Month Day, Year  
**Author:** Your Name

---

## Introduction

Write your opening paragraph here...

### Main Section

Write your main content here...

**Key points:**
- Point one
- Point two
- Point three

### Conclusion

Wrap up your thoughts...

---

*For more information, contact us at export@chateaualaverdi.ge*
```

---

## Tips

✅ **DO:**
- Write naturally, like you're telling a story
- Use headings to organize your content
- Keep paragraphs short and readable
- Add images to make posts visual
- Proofread before publishing

❌ **DON'T:**
- Forget the date in the filename (YYYY-MM-DD)
- Use spaces in the filename (use hyphens instead)
- Forget the `.md` extension

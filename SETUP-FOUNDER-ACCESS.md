# Setting Up Founder Access to Add Blog Posts

## Overview

## STEP 1: Add Founder as Collaborator (You Do This)

### Instructions for Repository Owner:

1. **Go to your GitHub repository:**
   ```
   https://github.com/gcharris/chateau-alaverdi-site
   ```

2. **Click "Settings"** (top navigation bar)

3. **Click "Collaborators"** (left sidebar under "Access")
   - You may need to confirm with your password

4. **Click "Add people"** (green button)

5. **Enter the founder's information:**
   - **If they have a GitHub account:** Enter their username
   - **If they don't:** Enter their email address (they'll be invited to create an account)

6. **Select their permission level:**
   - Choose **"Write"** (allows adding files but not changing settings)

7. **Click "Add [name] to this repository"**

8. **They'll receive an email invitation**

---

## STEP 2: Founder Accepts Invitation

### Founder needs to:

1. **Check email** for GitHub invitation

2. **Click "Accept invitation"** in the email

3. **If no GitHub account exists:**
   - Click link to create account at `https://github.com/signup`
   - Choose a username (e.g., `georgealverdi`)
   - Enter email and create password
   - Verify email address

4. **Once logged in:**
   - Visit: `https://github.com/gcharris/chateau-alaverdi-site`
   - You should see "You have write access to this repository"

---

## STEP 3: Adding Blog Posts (Founder Does This)

### Quick Reference for Founder:

**To add a new blog post:**

1. Go to: `https://github.com/gcharris/chateau-alaverdi-site`
2. Click `blog` ‚Üí `_posts`
3. Click "Add file" ‚Üí "Create new file"
4. Name file: `2025-11-15-my-post-title.md` (use today's date)
5. Write content using the template
6. Scroll down and click "Commit new file"
7. Post appears on website in 1-2 minutes!

**Full detailed instructions are in:**
```
blog/INSTRUCTIONS-FOR-FOUNDER.md
```

**Sample post to reference:**
```

```

## Troubleshooting

**If founder doesn't receive invitation email:**
- Check spam folder
- Resend invitation from GitHub settings
- Try using a different email address

**If founder can't access repository:**
- Verify they accepted the invitation
- Check they're logged into correct GitHub account
- Confirm you added them as "Write" collaborator

**If blog posts don't appear on website:**
- Wait 2-3 minutes for GitHub Pages to rebuild
- Check filename follows format: `YYYY-MM-DD-title.md`
- Ensure file is in `blog/_posts/` folder
- File must end with `.md` extension
